The DOMinator Webscraper

The DOMinator makes webscraping accessible to every member of your organisation, even non-technical staff

What is the DOM? It stands for Document-Object-Model. It's the tree like structure that makes up HTML pages.

Basic Concepts

The DOMinator works with configurations. Users can create, modify and delete these. The configurations specify what to scrape where and when.

A configuration is made up of three elements.
  • The name, which is used only for displaying that configuration to the user.
  • The URL, which determines which site to scrape.
  • And the ScrapeConfig, which determines what to scrape.

ScrapeConfig

The DOMinator uses a proprietary parsing language known as DOMScript. It makes specifying which elements you want easy and accessible.

DOMScript is made up of two main elements: TextQueries and TableQueries

Independant of the type, each query has a name and an object_name. The name is used only for displaying the query to the user, the object name however specifies the name of the property, under which the scraped data is saved.

TextQueries

A TextQuery allows you to select an Element using DOMScript and it will get the text content of that element and save it.

TableQueries

A TableQuery allows you to scrape table-like data and save it in an appropriate format. A TableQuery is made up of a TableRow query and the TableColumn queries. The TableRow query specifies which elements contain the rows of the table. They might be <li></li> elements or a div. The TableColumn queries specify which elements to get from each of the TableRows.

DOMScript

Inside Table- or TextQueries the user can make refinements to their query with the three other queries provided by DOMSCript. These can only be used inside a Table- or TextQuery to make more precise queries possible, for more complicated DOM structures.

BaseQuery

This is the simplest of the three queries. It simply takes the first element that corresponds to the query.
DOM:
<div>
<div class="girl">
Anna
</div>
<div class="boy">
Marcus
</div>
<div class="boy">
Alexander
</div>
</div>
Query:
TextQuery > BaseQueryAll (".boy")
Result:
<div class="boy">
Marcus
</div>

This Query would scrape the element with Marcus in it, as it's the first that matches the query.

BaseQueryAll

What if you wanted Alexander however? The BaseQueryAll makes it possible to specify that you want the first or the tenth, etc... element.
DOM:
<div>
<div class="girl">
Anna
</div>
<div class="boy">
Marcus
</div>
<div class="boy">
Alexander
</div>
</div>
Query:
TextQuery > BaseQueryAll (".boy" 1)
Result:
<div class="boy">
Alexander
</div>

This Query would scrape the element with Alexander in it, as it's the second (or first, we begin to count with 0 in the industry) element.
